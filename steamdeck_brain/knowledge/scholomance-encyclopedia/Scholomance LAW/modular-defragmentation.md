Modular Defragmentation

Purpose:
Break dense, intricate, interlocking concepts into clean, obtainable pieces so development accelerates through understanding.

Default Output Shape
## Summary
Plain-language definition of the concept.

## Core Knot
The main confusing, tangled, or overloaded part.

## Modular Breakdown
Named pieces of the concept, each with a clear job.

## Dependency Map
What depends on what, and what must exist first.

## Mechanism Flow
A → B → C explanation of how the system actually works.

## Strong Metaphor
One structurally accurate metaphor that clarifies the mechanism.

## Practical Translation
What this means for code, architecture, design, prompts, or implementation.

## Understanding Checkpoints
Small tests/questions that prove the concept is understood.

## Risks of Misunderstanding
Where the metaphor, model, or implementation could mislead us.

## Next Step
The smallest useful action to move the concept forward.
Usage Command

You can invoke it like:

Use Modular Defragmentation on this:
[paste concept/code/system/prompt]

or

Modular Defrag this architecture.

or

Break this down with Modular Defragmentation.
Operating Rule

Metaphors must function like load-bearing bridges, not wallpaper.

A metaphor is allowed only when it maps cleanly to the concept’s structure. If the metaphor does not clarify mechanism, dependency, or flow, it gets cut.
