# Implementation Plan: Scholomance Fairly Odd Wand System & UX Test-Drive

Implement a bounded mathematical formula authoring system (the **Scholomance Fairly Odd Wand**) that allows an AI agent (or the user) to submit structured visual proposals. These proposals are validated fail-closed against a strict schema, resolved deterministically, and rendered dynamically via a scene-side role dispatcher without per-asset scene code. 

Additionally, build a stunning, premium interactive **Wand UX Test-Drive** page (`/wand`) where users can author, edit, validate, and preview formula-driven assets interactively.

---

## User Review Required

> [!IMPORTANT]
> **Depth Limit Enforcement:** The validator caps recursive `composite` formula structures at exactly `4` levels of depth to guarantee bounded runtime and prevent stack overflows.
>
> **Self-Describing Presets:** The catalog IDs generated by the idempotent registrar are derived deterministically using FNV-1a hashing on `(role, formula bytes, sourceIntentHash)`.
>
> **Visual Dispatch Architecture:** The scene renders coordinates using registered `RoleBindings` in a dedicated dispatcher, allowing the AI to author completely new visual configurations without touching canvas render classes.

## Proposed Changes

### 1. Schemas & Presets
#### [NEW] [formula.schema.json](file:///home/deck/Desktop/Scholomance-V12-main/presets/schemas/formula.schema.json)
- Define a discriminated union JSON schema for the six leaf dialects (`parametric_curve`, `edge_trace`, `fractal_iter`, `grid_projection`, `fibonacci`, `template_based`) and the recursive `composite` dialect.
- Enforce strict parameter bounds and `additionalProperties: false`.

#### [NEW] [formula-proposal.schema.json](file:///home/deck/Desktop/Scholomance-V12-main/presets/schemas/formula-proposal.schema.json)
- Define the proposal container schema enclosing the formula with AI metadata (`rationale`, `confidence`, `reviewRequired`, `sourceIntentHash`, `evalSuiteId`).

### 2. Core Logic & Validation
#### [NEW] [formula-validator.js](file:///home/deck/Desktop/Scholomance-V12-main/codex/core/modulation/planner/formula-validator.js)
- Implements strict fail-closed validation of formulas and proposals.
- Enforces depth capping for composites at `MAX_DEPTH = 4`.
- Emits clean validation error arrays and the `FORMULA_PROPOSAL_REJECTED` warning when validation fails.

#### [NEW] [compose-formula.js](file:///home/deck/Desktop/Scholomance-V12-main/codex/core/modulation/processors/compose-formula.js)
- The `compose.formula.v1` processor corresponding to the `compose-formula` modulation stage.
- Takes incoming proposals, runs the validator, transforms unit-space coordinates to world-space, evaluates points via `evaluateFormula`, and tags coordinates with their `role` and `material`.

#### [NEW] [formula-registrar.js](file:///home/deck/Desktop/Scholomance-V12-main/codex/core/modulation/planner/formula-registrar.js)
- Provides deterministic registry persistence to `presets/proposed-formulas.json`.
- Implements deterministic FNV-1a hashing for unique, duplicate-free `catalogId` derivation.

### 3. Rendering Dispatcher
#### [NEW] [roleDispatcher.ts](file:///home/deck/Desktop/Scholomance-V12-main/src/ui/features/mysticHolistics/hero/roleDispatcher.ts)
- A scene-side registry that maps roles to rendering layers and drawers.
- Groups active frame coordinates by role and invokes drawers.

#### [NEW] [roleDrawers.ts](file:///home/deck/Desktop/Scholomance-V12-main/src/ui/features/mysticHolistics/hero/roleDrawers.ts)
- Implement rendering engines for standard roles (e.g. `shrine.altar`, `sigil.capsule`, `shrine.cabinet`, `shrine.window`, etc.) using modern high-acuity glow and atmospheric canvas rendering.

### 4. Interactive UX Test-Drive Website
