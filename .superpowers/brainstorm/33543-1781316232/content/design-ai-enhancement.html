Design — Section 4: AI Enhancement Flow

One-time bake. Claude Haiku looks at the finished character and tunes the 3 effects that matter most for that specific palette. Result stored permanently in the actor record.

Full flow — triggered once by "Forge & Enhance" button
Actor Forge Lab
→
bake(spec)
→
finalDT.snapshot()
→
base64 PNG
│
▼ POST /api/character/enhance
body: { imageDataUrl, schoolName, characterName }
— API key stays server-side (matches existing /api/combat/score pattern)
│
▼ server/routes/character-enhance.js → Anthropic SDK → Claude Haiku
model: claude-haiku-4-5-20251001 (vision, fast, cheap)
max_tokens: 128 (JSON only, no prose)
system: "Return only a JSON object matching the schema. No explanation."
message: [image, "Enhance effects for this {school} character sprite."]
│
▼ { glowIntensity, rimColor, atmosphereOpacity } ← validated + clamped
│
▼ pixelLotusActor.shaderEnhancements = result (localStorage)
│
▼ re-bake with enhanced uniforms merged → preview updated
AI output schema (3 values)
Field Type Clamp
glowIntensity float 0.3 – 1.5
rimColor "#rrggbb" valid hex, fallback to school primary
atmosphereOpacity float 0.0 – 0.8
Kept at 3 values (not 5) — the effects shader only exposes 3 tuneable knobs. rimColor overrides u_rimColor; the other two override their matching uniforms directly.
What Haiku actually sees + decides
Example: dark-robed Void school character
→ dark clothing, very dark bg
→ glowIntensity: 1.2 (push hard, needs contrast)
→ rimColor: "#a78bfa" (purple rim fits void palette)
→ atmosphereOpacity: 0.5 (prominent aura)
Example: light-robed Psychic school character
→ pale clothing, bright skin
→ glowIntensity: 0.6 (subtle, won't wash out)
→ rimColor: "#67e8f9" (cyan stays readable)
→ atmosphereOpacity: 0.25 (light touch)

Stored in actor record — applied every combat session

// localStorage key: pixelLotusActor
{
  spec: { ... },                 // CHARACTER-SPEC-v1
  displayName: "Lyria",
  shaderEnhancements: {       // null until Forge & Enhance runs
    glowIntensity: 1.2,
    rimColor: "#a78bfa",
    atmosphereOpacity: 0.5
  }
}

If shaderEnhancements is null, base effects uniforms (derived from school palette) are used as-is. Re-runnable: user can re-enhance any time, new result overwrites.
