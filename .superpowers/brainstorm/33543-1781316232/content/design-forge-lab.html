Design — Section 6: Actor Forge Lab Changes

Two additions to the existing lab UI: a "Forge & Enhance" button that runs the AI bake, and a before/after preview panel that shows the result.

Revised Actor Forge Lab — bottom action area
BASE
school defaults
ENHANCED ✦
AI-tuned effects
"Enter the Void" works without enhancing — uses school defaults

"Forge & Enhance" button state machine

idle
→ click →
forging…
→ bake done →
enhancing…
→ AI returns →
enhanced ✦
forging… — bake() + effects shader running (fast, ~100ms)
enhancing… — POST /api/character/enhance in flight (Haiku ~1-2s)
enhanced ✦ — shaderEnhancements stored, enhanced preview rendered, re-enterable
error state — network/API failure → toast "Enhancement failed, using defaults" → falls back gracefully

What changes in ActorForgeLab.tsx

enhancementState 'idle' | 'forging' | 'enhancing' | 'enhanced' | 'error' — drives button label + preview layout
enhancedPreviewUrl string | null — data URL of the AI-enhanced bake shown in the right preview panel
handleForgeAndEnhance() async fn: bake → snapshot → POST enhance → store → re-bake enhanced → set enhancedPreviewUrl
handleEnterVoid() unchanged — stores pixelLotusActor (now includes shaderEnhancements if present) → navigate /combat
