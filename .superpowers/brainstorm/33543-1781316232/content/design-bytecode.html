Design — Section 2: The Bytecode Format

How CHARACTER-SPEC-v1 fields compile to GLSL uniforms. This is the contract that makes the spec "executable."

character-bytecode-compiler.js — Uniform Contract
Uniform GLSL Type Source in SPEC How Derived
u_seed float spec.seed Direct. Drives noise variation in hair + cloth.
u_skinColor vec3 spec.materials.skin material-registry.js anchors.body → hexToVec3
u_hairColor vec3 spec.materials.hair material-registry.js anchors.body → hexToVec3
u_clothColor vec3 spec.clothing[top].profile clothing profile → material slot (cloth_linen, cloth_wool…)
u_schoolGlow vec3 spec.combatProfile.school SCHOOL_PALETTE[school].glow → hexToVec3
u_rimColor vec3 spec.combatProfile.school SCHOOL_PALETTE[school].primary → hexToVec3
u_eyeColor vec3 spec.materials.eyes material-registry.js anchors.body → hexToVec3
u_bodyWidth float spec.presentation.buildClass slender=0.8, average=1.0, stocky=1.2
u_bodyHeight float spec.presentation.heightClass short=0.85, average=1.0, tall=1.15
u_hairLength float spec.hair.profile bald=0.0, short=0.35, medium=0.65, long=1.0
u_hairStyle float spec.hair.profile int index: 0=short 1=medium 2=long 3=ponytail 4=curly 5=bald
u_eyeShape float spec.face[leftEye].profile 0=round 1=almond 2=narrow 3=voidTouched (affects SDF aspect ratio)
u_glowIntensity float spec.combatProfile.school VOID=0.9, PSYCHIC=0.8, SONIC=0.6, default=0.5
ShaderEnhancements — AI Output Schema (stored in pixelLotusActor)
Uniform Range Effect
u_glowIntensity 0.0 – 1.5 Overrides base glow. AI can push school energy higher.
u_contrastBoost -0.3 – 0.5 Post-process contrast. Makes silhouette pop in dark arena.
u_saturationShift -0.3 – 0.5 Shifts overall palette saturation toward school identity.
u_enhancedRim vec3 hex AI-chosen rim light color. Layered on top of base u_rimColor.
u_atmosphereOpacity 0.0 – 0.8 School-colored atmospheric haze around the character's feet.
Stored as: pixelLotusActor.shaderEnhancements — merged over base uniforms at render time.
Defaults to null (base uniforms used as-is) until "Forge & Enhance" is run.
Re-runnable: user can re-enhance at any time; new result overwrites stored value.

Determinism guarantee

Same CHARACTER-SPEC-v1 + same seed → identical uniform object every run. The compiler is a pure function with no randomness. The AI enhancement is the only non-deterministic element, and it is stored — so after the first "Forge & Enhance," combat rendering is fully deterministic again.
